# Coppice web UI

The cluster observability and debugging UI: queue health, cluster
capacity, per-job deep dives (timeline, spec, queue-position explainer,
accrual funding, cost/usage, logs), per-node deep dives, and a
coordinator debug page.

**Status: frontend template on mock data.** There is no dependency on a
running coordinator — every view is served by a deterministic in-browser
cluster simulation (`src/api/mock/`). Real endpoints replace the mock one
at a time as the coordinator's HTTP API is built; the seam is the
`CoppiceApi` interface in `src/api/client.ts` (see CLAUDE.md for the
exact procedure and the architecture rules).

## Running

```sh
cd web
npm install
npm run dev       # http://localhost:5173, mock data
npm run build     # typecheck + static bundle in dist/
npm test          # mock-world invariants + component smoke tests
npm run lint      # oxlint
npm run format    # prettier
```

Requires Node ≥ 20.

## Stack

- Vite, React 19, TypeScript (strict)
- TanStack Router (file-based routes in `src/routes/`) + TanStack Query
- Tailwind CSS v4 with vendored shadcn/ui-style primitives
  (`src/components/ui/`)
- Recharts for charts; Vitest + Testing Library for tests

## How this ships (future)

Coppice deploys as a single `coppice` binary, and the public API is
JSON-over-HTTP on every coordinator replica's `listen.client_addr`
(default `:7070`). When that listener exists, `web/dist` gets embedded
into the binary (e.g. `rust-embed`/`include_dir`) and served by the same
listener: static assets at `/`, SPA fallback to `index.html` for client
routes, and the JSON API under `/api/v1/...` — so the UI is same-origin
with the API and needs no CORS. Auth follows the API's SSO (ADRs
0022/0023); until then the UI is open and logged in as "Demo User".

## Repo etiquette

`web/` is intentionally self-contained: no Rust build integration yet.
CI runs lint/typecheck/test/build via the `web` job in
`.github/workflows/ci.yml`. `src/routeTree.gen.ts` is generated by the
router plugin (committed, never hand-edited).
